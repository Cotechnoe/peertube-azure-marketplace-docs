# Finalize the PeerTube Instance

**No action required.** The VM configures itself automatically on first boot.

## What happens automatically

On first boot, the systemd service `peertube-firstboot.service` runs automatically and performs these steps:

1. Determines the VM's public hostname (via Azure IMDS or the Custom data field).
2. Generates a random PostgreSQL password and application secret (unless provided in Custom data).
3. Generates the PeerTube configuration file (`production.yaml`).
4. Creates the PostgreSQL database and role for PeerTube.
5. Generates a self-signed TLS certificate for HTTPS.
6. Configures and enables the nginx reverse proxy.
7. Installs and starts `peertube.service`.
8. Waits for the application to respond on port 443.

All credentials are generated on the VM at runtime — they are never stored in the image.

Finalization takes approximately **3 to 8 minutes** after the VM starts.

## Customize the deployment (optional)

When creating the VM, the **Advanced** tab in the Azure portal provides a **Custom data** field. You can enter `key=value` parameters to override the defaults:

```
peertube_hostname=peertube.example.com
peertube_admin_email=admin@example.com
peertube_db_password=MyDBPassword
peertube_secret=MyApplicationSecret64chars
```

| Key | Description | Default if absent |
|-----|-------------|-------------------|
| `peertube_hostname` | Domain name or public IP to access PeerTube | Public IP detected via Azure IMDS |
| `peertube_admin_email` | PeerTube super-administrator email | `admin@<hostname>` |
| `peertube_db_password` | PostgreSQL password | Randomly generated (`openssl rand`) |
| `peertube_secret` | PeerTube application secret | Randomly generated (`openssl rand`) |

> All parameters are **optional**. If the Custom data field is empty, the VM configures itself automatically with securely generated values.

## Retrieve the admin password

The `root` PeerTube account password is generated by PeerTube itself on first startup. Connect to the VM via SSH and retrieve it from the system journal:

```bash
sudo journalctl -u peertube.service --no-pager | grep "User password"
```

Example output:
```
Jun 14 20:12:57 my-vm node[1910]: User password: <generated-password>
```

Save this password — you will use it to log in to the PeerTube web interface as `root`.

## Verify the instance

Open a browser and navigate to `https://<public-ip-or-domain>`.

- With a public IP or self-signed certificate, your browser will display a security warning. This is expected — click "Advanced" and proceed.
- With a domain name and Let's Encrypt configured, the certificate will be valid.

Check that you can:

1. See the PeerTube home page.
2. Log in as `root` with the password retrieved above.
3. Navigate to **Administration > Overview** to confirm the server is running.

## Monitor finalization progress (optional)

To follow the finalization logs in real time:

```bash
sudo journalctl -u peertube-firstboot.service -f
```

## Troubleshooting

| Problem | Cause | Solution |
|---------|-------|----------|
| PeerTube not accessible 10 minutes after boot | Firstboot failure | Check `sudo journalctl -u peertube-firstboot.service -n 50` |
| PeerTube configured with a private IP (e.g. `10.x.x.x`) | IMDS did not return the public IP | Connect via SSH and re-run manually: `sudo PEERTUBE_HOSTNAME=<public-ip> PEERTUBE_FORCE=1 /usr/local/sbin/finalize-peertube-vm.sh` |
| Home page not loading | Service startup issue | `sudo systemctl status peertube.service` and `sudo journalctl -u peertube.service -n 50` |
| Browser certificate error | Self-signed certificate | Expected — click "Advanced" and proceed, or configure a domain with Let's Encrypt |
| Cannot retrieve admin password | Logs not yet written | Wait 30 seconds and retry the `journalctl` command |
